## САОД. Задание 3: Префиксное дерево для решения задачи подсчета слов.

### Постановка задачи

Имеется большой ASCII-файл (дамп статей из Википедии), который рассматривается как одна строка и представлен в строковом объекте. Рассматривается задача подсчёта числа вхождений каждого слова в строке, то есть составления ассоциативного контейнера вида (слово → число вхождений слова в строку) с использованием префиксного дерева.

Под словом понимается последовательность символов из множества [a-zA-Z'-]. Порядок символов и их количество не ограничиваются; это также относится к символам апострофа и дефиса.

Необходимо реализовать структуру данных префиксное дерево для хранения словаря слов и количества их вхождений в тексте. Реализацию желательно оформить в виде контейнера с методами; интерфейс (абстрактный класс) представлен ниже.

После реализации требуется сравнить время формирования словаря, реализованного с помощью префиксного дерева, со временем формирования словаря, реализованного с использованием хеш-таблицы (пример приведён ниже).

Результаты вычислительных экспериментов по измерению времени необходимо представить с помощью диаграммы размаха (boxplot), выполнив несколько запусков программы (не менее 10 раз) и измерив время формирования словаря.

Число вхождений каждого слова в словаре, а также общее число уникальных слов должны совпадать со значениями, полученными при использовании хеш-таблицы из приведённого примера или собственной реализации.

Например, число вхождений слова `Dubna` в Вашем словаре и в хеш-таблице должно быть одинаковым при идентичном способе выделения слов при проходе по строке. Проверку корректности работы вашего словаря необходимо оформить в функции с аналогичной сигнатурой.

Для формирования словаря допускается только один проход по тексту, который предварительно загружен в оперативную память и хранится там в явном виде в виде одной большой строки.

Реализуемая структура должна поддерживать:

* добавление слова в словарь;
* получение числа вхождений заданного слова;
* получение количества уникальных слов в словаре (за константное время);
* получение числа узлов префиксного дерева (за константное время).

### Подсчет вхождения слова

Рассмотрим число вхождений слова `Dubna` в файле `engwiki-ascii-20260201_1gb.txt`. Для этого воспользуемся командой `grep`, предварительно убедившись, что файл содержит только ASCII-текст. Файл представлен в корне репозитория в папке `data`.

```
[~]$ file engwiki-ascii-20260201_1gb.txt 
engwiki-ascii-20260201_1gb.txt: ASCII text, with very long lines (1599)
[~]$ grep -o -E '\<Dubna\S*' engwiki-ascii-20260201_1gb.txt | sort | uniq -c | sort -nr
     72 Dubna
     38 Dubna,
      6 Dubna.
      4 Dubna-Livermore
      2 Dubna;
      1 Dubna's
      1 Dubna";
```
Обратите внимание, что слово `Dubna` по вышеописанному определению слова входит в текст только 119 раз: 72 + 38 + 6 + 2 + 1. Слова `Dubna-Livermore` и `Dubna's` по указанному определению являются другими словами.

Получить число 119 можно, например, используя команду 
```
[~]$ perl -nle '$count++ while /\bDubna\b(?![-'"'"'])/g; END{print $count}' engwiki-ascii-20260201_1gb.txt
```
или используя контейнер `unordered_map`, предварительно загрузив файл целиком
```C++
#include <iostream>
#include <string>
#include <fstream>
#include <string_view>
#include <unordered_map>
#include <chrono>
#include <cctype>

void umap(std::string_view text, std::string_view word) {
 std::unordered_map<std::string_view, size_t> dict;

 const char* data = text.data();
 size_t n = text.size();

 size_t start = 0;
 bool in_word = false;

 for(size_t i = 0; i < n; ++i) {
  auto ch = data[i];
  bool is_letter = std::isalpha(ch) || ch == '\'' || ch == '-';

  if(is_letter) {
   if(!in_word) {
    start = i;
    in_word = true;
   }
  }
  else if(in_word) {
   std::string_view sv(data + start, i - start);
   ++dict[sv];
   in_word = false;
  }
 }

 if(in_word) {
  std::string_view sv(data + start, n - start);
  ++dict[sv];
 }
 std::cout << "dict size: " << dict.size() << std::endl;
 std::cout << word << ": " << dict[word] << std::endl;
}

int main() {
 std::string text;
 std::ifstream fin("engwiki-ascii-20260201_1gb.txt", std::ios::binary);
 if(!fin.is_open()) {
  std::cout << "not open!" << std::endl;
  return 0;
 }
 text.append((std::istreambuf_iterator<char>(fin)), std::istreambuf_iterator<char>());

 std::string word = "Dubna"; // слово, которое ищем
  
 const auto time_one{std::chrono::high_resolution_clock::now()};
  
 umap(text, word);
  
 const auto time_two{std::chrono::high_resolution_clock::now()};
 std::cout << "time: " << std::chrono::duration<double>(time_two - time_one).count() << " seconds" << std::endl;
}
// clang++ -std=c++17 -O3 -stdlib=libc++ example.cpp
// ./a.out
// dict size: 1428994
// Dubna: 119
// time: 8.43518 seconds
```

### Общие требования

Допускается реализация на любом компилируемом языке программирования общего назначения. Важно, чтобы реализация префиксного дерева по времени работы превосходила классическую хеш-таблицу с методом цепочек, доступную (или реализуемую) стандартными средствами выбранного языка или реализованную.

Запрещается использование многопоточного программирования, а также явное применение внешних технологий параллельной обработки (например, SIMD). Допускается только один линейный проход по строке. Ограничений на использование оперативной памяти нет. Предварительное резервирование памяти перед формированием словаря не запрещено, но время, затраченное на эту операцию, подлежит обязательному учету при замерах.

### Требования для реализации на C++

В случае реализации на языке программирования `C++` класс должен наследоваться от следующего абстрактного класса и переопределять все его виртуальные методы:
```C++
struct ITrie {
  virtual ~ITrie() = default;
  virtual void insert(std::string_view str) = 0;
  virtual size_t get(std::string_view str) const = 0;
  virtual size_t nodes() const = 0;   // должно выполняться за константное время
  virtual size_t size() const = 0;    // должно выполняться за константное время
};
```

### Проверка корректности

Критерием проверки корректности является количество вхождений слова `Dubna`. Ожидаемые значения:

* 119 раз в файле `engwiki-ascii-20260201_1gb.txt`;
* 17 раз в файле `simplewiki-20260201.txt`;
* 0 раз в строке `are they the most fun and these are a fun`.

Дополнительным необходимым критерием является количество уникальных слов и узлов префиксного дерева:

* в файле `engwiki-ascii-20260201_1gb.txt`: 1428994 слов и 4836427 узла;
* в файле `simplewiki-20260201.txt`: 610625 слов и 1939049 узла;
* в строке `are they the most fun and these are a fun`: 8 слов и 19 узла.

### Критерий оценки

Оценка определяется по среднему ускорению, полученному по результатам 10 запусков для файла `engwiki-ascii-20260201_1gb.txt`. Ускорение вычисляется как отношение среднего времени формирования словаря при использовании хеш-таблицы с методом цепочек к среднему времени формирования словаря при использовании префиксного дерева.

* 25 баллов при среднем ускорении до 0.75x
* 50 баллов при среднем ускорении в диапазоне [0.75x, 1.0)
* 75 баллов при среднем ускорении в диапазоне [1.0x, 1.25x)
* 100 балоов при среднем ускорении от 1.25x и выше
