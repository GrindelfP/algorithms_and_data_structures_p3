# Проект по дисциплине «Структуры и алгоритмы обработки данных»

## Требования к программной части проекта

### 1. Язык программирования

Проект должен быть реализован на языке **C** или **C++**. Выбор стандарта языка (например, C11, C17, C++17, C++20 и т.п.) осуществляется самостоятельно. Использование других языков программирования для реализации алгоритмов и структур данных **запрещено**.

### 2. Использование библиотек

Разрешается использование только стандартной библиотеки языка:

- для **C** — стандартной библиотеки языка C;
- для **C++** — стандартной библиотеки **STL**.

Использование **сторонних библиотек** запрещено без предварительного согласования.

### 3. Платформенная независимость

Код проекта должен быть **платформенно независимым**.

Запрещается использование платформозависимых функций и расширений компиляторов, в частности:

- функций и типов, характерных только для отдельных операционных систем;
- нестандартных расширений стандартной библиотеки;
- функций вида `strcpy_s`, `scanf_s` и аналогичных платформенных расширений.

### 4. Система сборки

Допускается использование любой удобной системы сборки, включая, но не ограничиваясь:

- `Makefile`
- `CMake`
- `Meson`
- встроенные системы сборки IDE

Система сборки должна позволять **полностью собрать проект из исходных кодов**.

### 5. Структура проекта

Структура проекта определяется студентом самостоятельно.

Разрешается:

- разделение проекта на несколько модулей;
- использование **статических** и **динамических библиотек**;
- наличие нескольких исполняемых файлов в составе проекта (например, для тестирования или демонстрации работы алгоритмов).

### 6. Ограничения на размер файлов

Размер исходных файлов должен быть разумным и поддерживать читаемость проекта.

Запрещается создание файлов исходного кода (`.c`, `.cpp` и т.д.) и заголовочных файлов (`.h`, `.hpp` и т.д.), размер которых превышает **1000 строк**.

При необходимости код должен быть разделён на несколько логически связанных модулей.

### 7. Особенности реализации

- при разработке проекта на языке **C++** допускается использование **модулей C++**;
- допускается использование **многопоточности** (например, threads.h или `std::thread`);
- использование **MPI** и других средств **распределённых вычислений** запрещено;
- использование **GPU-вычислений** и соответствующих технологий (например, **CUDA**, **OpenCL**) запрещено;
- использование **параллельных библиотек и фреймворков**, реализующих автоматическое распараллеливание алгоритмов (например, **OpenMP**, **TBB** и аналогичных средств), запрещено.

Использование иных средств параллельных или распределённых вычислений допускается только по согласованию.

### 8. Требования к читаемости кода

Исходный код должен соответствовать следующим требованиям:

- наличие осмысленных имен переменных, функций и структур;
- соблюдение единообразного стиля форматирования;
- отсутствие неиспользуемого кода;
- корректное разделение интерфейса и реализации.

### 9. Компиляция проекта

Проект должен успешно компилироваться без ошибок.  
Желательно устранить предупреждения компилятора или свести их количество к минимуму.

При сдаче проекта необходимо предоставить:

- полный исходный код;
- файлы системы сборки;
- инструкции по компиляции и запуску программы.

### 10. Документирование исходного кода

Исходный код проекта должен содержать документирующие комментарии в формате **Doxygen**.

Обязательному документированию подлежат:

- все публичные функции;
- структуры и классы;
- основные алгоритмы и используемые структуры данных;
- интерфейсы библиотечных модулей.

Документация должна включать:

- краткое описание назначения функции или структуры (обязательно);
- описание параметров через `@param` (необязательно);
- описание возвращаемого значения через `@return` (необязательно);
- при необходимости — краткое описание алгоритма и его сложности.

Пример оформления:

```cpp
/**
 * @brief Выполняет поиск элемента в бинарном дереве.
 *
 * @param root Указатель на корень дерева.
 * @param key Искомый ключ.
 * @return Указатель на найденный узел или nullptr, если элемент отсутствует.
 */
Node* find(Node* root, int key);
```
При документировании допускается использование английского языка (частично или полностью).

### 11. Тестирование

Проект должен содержать базовые тесты, демонстрирующие корректность реализации алгоритмов.

Тесты должны:

- проверять основные сценарии работы алгоритмов;
- включать граничные случаи;
- подтверждать корректность работы структур данных.

Допускается реализация тестов в виде отдельной программы или набора программ. Покрытие тестами всей реализации не требуется.

### 12. Требования к оформлению проекта

Проект должен содержать:

- файл `README.md` с общим описанием проекта;
- описание структуры каталогов;
- инструкции по сборке;
- инструкции по запуску программы.

## Требования к пояснительной записке

Помимо файла `README.md`, входящего в состав проекта, обязательно требуется подготовка **пояснительной записки**, содержащей теоретическое описание выполненной работы.

Пояснительная записка должна включать:

- описание реализованных алгоритмов и структур данных;
- краткое теоретическое обоснование выбранных решений;
- анализ вычислительной сложности основных реализованных алгоритмов.

Для каждого основного алгоритма необходимо указать:

- асимптотическую временную сложность (например, с использованием нотации **Big-O**);
- пространственную сложность (затраты памяти).

Желательно дополнительно провести **эмпирический анализ производительности**.

При возможности рекомендуется использовать методы анализа производительности, например **roofline-модель** или другие методы оценки вычислительной эффективности.

Допускается проведение анализа с использованием профилировщиков и средств измерения производительности на небольших тестовых примерах.

Допускается использование любых инструментов для **визуализации результатов работы алгоритмов**, включая внешние программы и библиотеки.

**Обязательное требование:** пояснительная записка должна быть оформлена в **LaTeX**. Для подготовки документа необходимо использовать предоставленный шаблон. Шаблон пояснительной записки [доступен в Overleaf](https://www.overleaf.com/read/pzdhjknwxksp#5a94c1).
